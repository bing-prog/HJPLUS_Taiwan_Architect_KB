# Archicad 基礎工作流程

## 為什麼重要？

Archicad 是建築設計常用的 BIM 軟體。若專案一開始沒有建立清楚的樓層、座標、圖層、視圖與出圖架構，後續會影響模型管理、圖面產出、顧問協調與 IFC 交換品質。

## 核心概念

### 1. 專案啟動

- 設定專案原點與定位基準。
- 建立樓層高度與樓層名稱。
- 建立軸網、基地參考與主要高程。
- 確認單位、尺寸標註、圖框與出圖比例。
- 優先使用事務所樣板，不要每案從空白檔開始。

### 2. 模型建立

建模順序建議由大到小：

1. 量體與樓層
2. 牆、版、柱、樑、屋頂
3. 門窗、樓梯、欄杆
4. 空間分區與房間資料
5. 重要節點與局部詳圖

細部不是全部都要用 3D 模型完成。施工詳圖、標註說明與局部放大，通常可以使用 Detail 或 Worksheet 輔助。

### 3. 視圖與圖面

Archicad 的文件產出可理解成四個層級：

| 層級 | 功能 |
|---|---|
| Project Map | 模型中的原始平面、立面、剖面、3D 視點 |
| View Map | 儲存比例、圖層、筆色、顯示方式後的正式視圖 |
| Layout Book | 圖紙、圖框、圖號、標題與版面配置 |
| Publisher Sets | 批次輸出 PDF、DWG、BIMx 或協調檔案 |

### 4. 協同作業

- 顧問模型需定期更新，並確認座標與樓層基準一致。
- 每次出圖前需檢查圖層組合、筆色、比例、圖框資訊與修訂紀錄。
- 大型專案應建立固定的模型檢查節奏，例如每週一次模型協調。

## 常見檢查清單

- 樓層高度是否與結構圖一致？
- 軸網是否與結構顧問一致？
- 門窗編號是否重複？
- Zone 面積是否合理？
- 圖紙日期、版次與圖號是否正確？
- PDF 輸出比例與線寬是否符合出圖標準？
