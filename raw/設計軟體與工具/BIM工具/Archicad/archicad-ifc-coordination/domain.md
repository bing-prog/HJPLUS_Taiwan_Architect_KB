# Archicad IFC 協調

## 為什麼重要？

IFC 是 BIM 軟體之間交換模型資訊的通用格式。建築師使用 Archicad 與結構、機電、營造或審查單位協調時，常會透過 IFC 交換模型。若座標、分類、屬性或匯出範圍設定不清楚，模型可能會錯位、缺件、變形或資訊遺失。

## IFC 交換應包含什麼？

| 資訊 | 範例 |
|---|---|
| 幾何 | 牆、版、門窗、樓梯、空間、屋頂 |
| 分類 | 元件類型、BIM 分類、IFC 類別 |
| 屬性 | 防火時效、隔音性能、材料、編號、階段 |
| 空間結構 | 基地、建築物、樓層、空間 |
| 定位 | 專案原點、高程、座標基準 |

## 匯出前檢查

- 專案原點與顧問約定座標是否一致？
- 樓層高度與高程基準是否一致？
- 是否建立專門的 IFC 匯出視圖？
- 不需要匯出的圖層是否已排除？
- 元件 ID、分類與屬性是否足以讓顧問辨識？
- Zone 或房間資訊是否需要納入？

## 顧問模型檢查

收到顧問 IFC 後，建議在協調檔中檢查：

1. 平面是否對位。
2. 剖面高程是否一致。
3. 結構柱、樑、版與建築空間是否衝突。
4. 機電管線是否穿越主要結構或天花高度。
5. 模型是否過重，是否含有不必要細節。
6. 元件是否有足夠分類與屬性。

## 常見問題

| 問題 | 可能原因 |
|---|---|
| 模型位置很遠 | 專案原點、測量座標或匯出座標設定不同 |
| 元件不見 | 圖層、樓層、翻修狀態或匯出篩選排除了元件 |
| 元件類型錯誤 | Archicad 分類或 IFC 類別映射不正確 |
| 檔案過大 | 匯出過多細節、物件或使用過於精確的幾何 |
| 材料資訊錯誤 | 建材屬性或 IFC Property Mapping 設定不完整 |

## 建議交付內容

- 有日期的 IFC 檔。
- 說明匯出範圍的 PDF 或圖片。
- 坐標基準說明。
- 使用的 IFC Translator 名稱。
- 與前一版相比的變更摘要。
- 協調問題清單與責任分工。
