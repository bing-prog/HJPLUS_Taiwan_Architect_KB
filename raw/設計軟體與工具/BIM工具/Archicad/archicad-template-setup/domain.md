# Archicad 樣板設定

## 為什麼重要？

Archicad 樣板是事務所 BIM 標準的核心。好的樣板可以讓每個專案從一致的圖層、筆色、構造、視圖、圖框與出圖設定開始，減少重複設定，也降低圖面風格不一致與模型混亂的風險。

## 樣板應包含什麼？

| 項目 | 用途 |
|---|---|
| 樓層設定 | 建立常用樓層命名與高度邏輯 |
| 圖層與圖層組合 | 控制模型元素顯示與出圖狀態 |
| 筆色與線寬 | 控制圖面階層、線寬與列印效果 |
| 建材 | 控制材料名稱、交接優先序與剖面表現 |
| 複合構造 | 建立常用牆、版、屋頂與裝修構造 |
| 收藏夾 | 儲存常用牆、門窗、標註、物件設定 |
| 視圖 | 預設平面、立面、剖面、詳圖與表格 |
| 圖框 | 建立標準圖紙尺寸、圖框與圖號規則 |
| 發布集 | 批次輸出 PDF、DWG、BIMx 或協調檔案 |

## 建議命名原則

- 名稱應穩定、可搜尋、可排序。
- 不同專業或用途應有一致前綴。
- 避免同一材料出現多個近似名稱，例如「RC」、「鋼筋混凝土」、「混凝土-結構」混用。
- 樣板名稱不宜綁定單一專案，專案特有設定應留在專案檔內。

## 維護流程

1. 收集專案中反覆出現的設定問題。
2. 判斷是個案需求還是事務所共通需求。
3. 在測試檔中調整樣板。
4. 測試 PDF、DWG、IFC 輸出。
5. 更新樣板版本與變更紀錄。
6. 通知團隊新版本差異與使用注意事項。

## 常見風險

- 圖層太多，導致使用者不知道該放在哪一層。
- 筆色經常改動，造成舊專案出圖結果不穩定。
- 建材與複合構造缺乏命名規則，造成交接線錯誤。
- 樣板混入單一專案的特殊內容，導致下一個專案被污染。
